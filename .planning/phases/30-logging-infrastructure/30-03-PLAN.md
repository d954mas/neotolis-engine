Migrate all `printf()`/`fprintf(stderr, ...)` calls in builder source files to `NT_LOG_INFO()`/`NT_LOG_ERROR()` macros per D-04. Add `#include "log/nt_log.h"` to each file that gains log calls.

**Transformation rules:**

**Rule 1 - printf to NT_LOG_INFO:**
```c
// BEFORE:
(void)printf("Starting pack: %s\n", output_path);
// AFTER:
NT_LOG_INFO("Starting pack: %s", output_path);
```
Remove `(void)` cast, remove `\n` from format string (log system adds newline), change `printf` to `NT_LOG_INFO`.

**Rule 2 - fprintf(stderr, ...) to NT_LOG_ERROR:**
```c
// BEFORE:
(void)fprintf(stderr, "ERROR: No assets added.\n");
// AFTER:
NT_LOG_ERROR("No assets added");
```
Remove `(void)` cast, remove `\n` from format string, remove `"ERROR: "` prefix (level is auto-injected), change `fprintf(stderr, ...)` to `NT_LOG_ERROR(...)`.

**Rule 3 - fprintf(stderr, ...) with format args:**
```c
// BEFORE:
(void)fprintf(stderr, "ERROR: Failed to load %s\n", path);
// AFTER:
NT_LOG_ERROR("Failed to load %s", path);
```

**Rule 4 - printf with progress/stats info:**
```c
// BEFORE:
(void)printf("  Meshes: %u\n", count);
// AFTER:
NT_LOG_INFO("  Meshes: %u", count);
```
Keep the indentation in the format string for readability.

**Rule 5 - DO NOT MIGRATE `NT_BUILD_ASSERT`:**
The `NT_BUILD_ASSERT` macro in `tools/builder/nt_builder_internal.h` uses `fprintf(stderr, ...)` + `exit(1)`. This is an assertion mechanism, NOT a logging call. Do NOT modify it per research Pitfall 8.

**Rule 6 - main.c entry point:**
`tools/builder/main.c` is the CLI entry point. Its printf calls can use `NT_LOG_INFO`/`NT_LOG_ERROR` since the builder library has `NT_LOG_DOMAIN_DEFAULT="builder"` and main.c is compiled as part of the builder target (or linked against it). If main.c is compiled separately (as `add_executable(builder main.c)` without the domain definition), use plain functions `nt_log_info()`/`nt_log_error()` instead. Check the CMakeLists.txt: main.c links against `nt_builder` which provides the include path, but the `NT_LOG_DOMAIN_DEFAULT` is set on `nt_builder` target only. The `builder` executable target needs its own domain definition OR main.c should use plain functions. **Resolution:** Add `target_compile_definitions(builder PRIVATE NT_LOG_DOMAIN_DEFAULT="builder")` to the builder executable target in CMakeLists.txt, then use `NT_LOG_INFO`/`NT_LOG_ERROR` in main.c.
