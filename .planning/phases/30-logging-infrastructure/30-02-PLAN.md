  <action>
Migrate all `nt_log_info()`/`nt_log_error()` calls in engine modules to `NT_LOG_INFO()`/`NT_LOG_WARN()`/`NT_LOG_ERROR()` domain macros. Apply these transformation rules per D-01, D-02, D-03:

**Rule 1 - snprintf pattern removal:**
```c
// BEFORE:
char buf[128];
(void)snprintf(buf, sizeof(buf), "nt_resource: pack 0x%08X loaded (%u bytes)", pack->pack_id, loaded_size);
nt_log_info(buf);

// AFTER:
NT_LOG_INFO("pack 0x%08X loaded (%u bytes)", pack->pack_id, loaded_size);
```
Delete the `char buf[]` declaration, delete the `snprintf` call, replace `nt_log_info(buf)` with `NT_LOG_INFO(fmt, args)` using the format string and args from the deleted snprintf. Remove any manual domain prefix like `"nt_resource: "` or `"gfx: "` from the format string (domain is auto-injected by the macro).

**Rule 2 - Static string migration:**
```c
// BEFORE:
nt_log_error("gfx: backend init failed");
// AFTER:
NT_LOG_ERROR("backend init failed");
```
Remove manual domain prefix. Change `nt_log_error` to `NT_LOG_ERROR`.

**Rule 3 - String pass-through (GL info log):**
```c
// BEFORE (nt_gfx_gl.c):
nt_log_error(info_log);  // info_log is a char buffer from GL
// AFTER:
NT_LOG_ERROR("%s", info_log);  // Use %s to avoid format injection
```

**Rule 4 - Level mapping:**
- `nt_log_info(...)` -> `NT_LOG_INFO(...)` for informational messages
- `nt_log_error(...)` -> `NT_LOG_ERROR(...)` for error messages
- Some messages that are currently `nt_log_error` but describe warnings (e.g., "already mounted", "fallback used") -> consider `NT_LOG_WARN` if appropriate
