  <action>
    Replace stub implementations in nt_resource.c:

    **nt_resource_create_pack():**
    1. Check initialized, return NT_ERR_INVALID_ARG if not
    2. Check pack_id not already mounted (same check as mount), return NT_ERR_INVALID_ARG if duplicate
    3. Find free slot in packs[] (mounted==0), return NT_ERR_INVALID_ARG if full
    4. Set pack_id, priority, pack_type=NT_PACK_VIRTUAL, mounted=1, asset_start=0, asset_count=0, blob=NULL, blob_size=0
    5. Return NT_OK

    **nt_resource_register():**
    1. Check initialized, return NT_ERR_INVALID_ARG if not
    2. Find pack by pack_id (must be mounted and pack_type==NT_PACK_VIRTUAL), return NT_ERR_INVALID_ARG if not found or not virtual
    3. Check for duplicate: scan assets[] for matching resource_id + pack_index. If found, update runtime_handle and state=READY, set needs_resolve=true, return NT_OK
    4. Find next free slot in assets[] (resource_id == 0 for unused, or use asset_count if contiguous). For simplicity, scan from 0 to find a slot with resource_id==0 (freed by unregister/unmount). If none found and asset_count < NT_RESOURCE_MAX_ASSETS, use assets[asset_count] and increment asset_count.
    5. Populate NtAssetMeta: resource_id, asset_type, state=NT_ASSET_STATE_READY, format_version=0, pack_index=index of this pack, offset=0, size=0, runtime_handle=gfx_handle
    6. Update pack's asset_count. For virtual packs, asset_start tracks the first entry index. If this is the first entry for the pack, set asset_start. Otherwise, virtual pack entries may not be contiguous -- need to track differently.

    **Important design note for virtual pack entries:** Unlike file packs (which get contiguous AssetMeta entries from parse_pack), virtual pack entries are added one at a time and may not be contiguous. The pack's asset_start/asset_count scheme doesn't work for virtual packs with interleaved adds. Solutions:
    - Option A: Virtual packs don't use asset_start/asset_count. Instead, scan all assets[] for matching pack_index.
    - Option B: Use asset_count as a count but entries scattered -- unmount/unregister scans all assets[] for pack_index match.
    - Choose Option B: set asset_count to track total entries added (for stats), but unmount and resolve scan all assets[] checking pack_index. For file packs, the contiguous range [asset_start..asset_start+asset_count) is an optimization. For virtual packs, full scan is acceptable (at mount/unmount time, not per-frame).

    **nt_resource_unregister():**
    1. Check initialized, return if not
    2. Find pack by pack_id, return if not found
    3. Scan assets[] for entry with matching resource_id AND pack_index == pack index
    4. If found: set resource_id = 0 (mark as free), decrement pack's asset_count
    5. Set needs_resolve = true

    **nt_resource_set_placeholder():**
    1. Validate asset_type is 1, 2, or 3 (NT_ASSET_MESH, NT_ASSET_TEXTURE, NT_ASSET_SHADER_CODE)
    2. Store: `s_resource.placeholder_handles[asset_type] = gfx_handle`
    3. Set needs_resolve = true (in case slots currently using placeholder should update)

    **Update nt_resource_step() resolve logic** (from Plan 02):
    In the resolve loop, after scanning for best READY entry, if no READY entry found:
    1. Check `s_resource.placeholder_handles[slot.asset_type]`
    2. If non-zero: set slot.gfx_handle = placeholder_handle. Keep slot.state as-is (REGISTERED/LOADING/FAILED -- reflects actual state, not placeholder state)
    3. If zero: slot.gfx_handle = 0, state unchanged

    **Update nt_resource_unmount()** to handle virtual packs:
    - For NT_PACK_FILE packs: existing behavior (clear entries, set needs_resolve). GPU resource cleanup is Phase 25's responsibility (runtime_handle destruction).
    - For NT_PACK_VIRTUAL packs: clear entries (resource_id = 0) but do NOT destroy GPU handles. The game created them, the game destroys them. Log info message: "Virtual pack unmounted, game must destroy GPU handles."
    - In both cases: scan ALL assets[] for matching pack_index (not just asset_start range) to handle non-contiguous virtual pack entries.
