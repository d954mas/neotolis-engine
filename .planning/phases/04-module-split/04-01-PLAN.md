<!-- Key targets and their public APIs relevant to this plan -->

From engine/core/CMakeLists.txt (nt_core target):
```cmake
nt_add_module(nt_core nt_core.c nt_core.h nt_types.h nt_platform.h)
target_link_libraries(nt_core PUBLIC nt_shared)
```
nt_add_module() sets PUBLIC include dir as engine/, so consumers get
`#include "core/nt_core.h"` resolution automatically via linking nt_core.

From cmake/nt_module.cmake (nt_add_module behavior):
- Creates STATIC library
- Sets PUBLIC include directory: `${CMAKE_SOURCE_DIR}/engine`
- Applies warning flags, sanitizer flags, output directory

Existing targets available for consumer linking:
- `nt_core` -- STATIC, provides engine/core headers via PUBLIC include
- `nt_shared` -- INTERFACE, transitively available via nt_core PUBLIC link
- `cglm_headers` -- INTERFACE (header-only), from deps/cglm EXCLUDE_FROM_ALL
- `unity` -- STATIC, from deps/unity EXCLUDE_FROM_ALL
</interfaces>

**engine/CMakeLists.txt** -- Replace entire file with single line:
```cmake

```
Delete lines 3-11 (the nt_engine INTERFACE shim, transitional comments, and cglm_headers integration). Per user decision: no deprecation warning, no compatibility layer -- just remove it.

**examples/hello/CMakeLists.txt** -- Change line 2 from:
```cmake
target_link_libraries(hello PRIVATE nt_engine)
```
to:
```cmake
target_link_libraries(hello PRIVATE nt_core cglm_headers)
```
Per user decision: hello links cglm_headers even though its source does not currently include cglm. This is intentional future-proofing. cglm_headers is INTERFACE so it has zero binary impact.

**tests/CMakeLists.txt** -- Three consumer migrations:

1. test_core (line 6): change `nt_engine` to `nt_core`:
   ```cmake
   target_link_libraries(test_core PRIVATE nt_core unity)
   ```
   Remove `target_include_directories(test_core PRIVATE ${CMAKE_SOURCE_DIR}/engine)` on line 7 -- it is redundant because nt_core already provides engine/ as PUBLIC include via nt_add_module().

2. test_sanitizer_proof (line 25): change `nt_engine` to `nt_core`:
   ```cmake
   target_link_libraries(test_sanitizer_proof PRIVATE nt_core unity)
   ```
   Remove `target_include_directories(test_sanitizer_proof PRIVATE ${CMAKE_SOURCE_DIR}/engine)` on line 26 -- same redundancy as above.
   Per user decision: test_sanitizer_proof links nt_core even though its source does not directly include engine headers. This is intentional -- it proves sanitizers work on an executable that includes engine code in its link set.

3. wasm_smoke (line 42): change `nt_engine` to `nt_core`:
   ```cmake
   target_link_libraries(wasm_smoke PRIVATE nt_core)
   ```
   Remove `target_include_directories(wasm_smoke PRIVATE ${CMAKE_SOURCE_DIR}/engine)` on line 43 -- same redundancy.

**tools/builder/CMakeLists.txt** -- Change line 2 from:

to:
