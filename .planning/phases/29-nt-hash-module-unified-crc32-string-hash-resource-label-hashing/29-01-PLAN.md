    **engine/hash/nt_hash.c** -- Implementation with:
    - FNV-1a 32-bit: offset=0x811C9DC5U, prime=0x01000193U (byte-by-byte loop)
    - FNV-1a 64-bit: offset=0xCBF29CE484222325ULL, prime=0x00000100000001B3ULL (byte-by-byte loop)
    - Label table (only compiled when NT_HASH_LABELS=1):
      - `#define LABEL_TABLE_SIZE (NT_HASH_MAX_LABELS * 2)` (power-of-2 for fast modulo via & mask)
      - Static arrays `s_labels32[LABEL_TABLE_SIZE]` and `s_labels64[LABEL_TABLE_SIZE]`
      - Each entry: `{ uint32_t/uint64_t hash_value; char label[128]; }` (0 = empty slot)
      - Open-addressing linear probe using `hash.value % LABEL_TABLE_SIZE`
      - Register: find empty or matching slot, strncpy label (127 chars max + null)
      - Lookup: probe until empty or match, return label or NULL
      - init clears tables via memset, shutdown clears tables
    - When NT_HASH_LABELS=0: register functions are `(void)hash; (void)label;` no-ops, lookup returns NULL
    - `nt_hash_init`: memset label tables if NT_HASH_LABELS, return NT_OK
    - `nt_hash_shutdown`: memset label tables if NT_HASH_LABELS

    **tools/research/hash_benchmark/main.c** -- Standalone benchmark with:
    - All 4 candidates implemented locally (no external deps):
      1. **FNV-1a**: 32-bit (offset=0x811C9DC5, prime=0x01000193) and 64-bit (offset=0xCBF29CE484222325, prime=0x100000001B3)
      2. **xxHash**: xxh32 and xxh64 -- minimal implementation from public domain (or self-contained port). Use byte-by-byte fallback if block version is too large. Only needs to produce correct results for benchmarking.
      3. **MurmurHash3**: x86_32 (32-bit output) and x86_128 (take upper 64 bits for 64-bit). Use PeterScott public domain C port or reimplement.
      4. **CRC32-as-hash**: Use CRC32 lookup table (same as shared/nt_crc32.c). For 64-bit, XOR two halves of input hashed separately.
    - Benchmark loop:
      - Short strings: "textures/lenna.png" (20 chars), "assets/meshes/cube.glb" (22 chars) -- 1M iterations
      - Medium strings: 100-char path string -- 1M iterations
      - Measure wall-clock time via clock_gettime or QueryPerformanceCounter
      - Print: candidate name, width (32/64), input length, total ms, ops/sec
    - Avalanche quality: hash 10000 sequential strings ("key_0000" to "key_9999"), count unique 32-bit values, report collision count
    - Code size estimate: print approximate code size of each implementation (count lines or bytes)
    - Output results as plain text table to stdout
